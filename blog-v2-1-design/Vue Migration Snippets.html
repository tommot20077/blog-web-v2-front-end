§ React → Vue 3 · 用 <script setup> 寫

同樣的設計系統，換一套
框架住進去。

把 prototype 裡的 hooks 改寫成 Vue 3 composables、把 context provider 換成 provide/inject、把 className 換成 class。其實移動的東西不多，因為設計 token 都寫在 CSS。

§ 01 — Composables

useToast / useTheme / useReadingProgress

3 files

React 裡的 Provider + Hook 模式，在 Vue 3 用 provide + inject 加一個 composable 就能對應。下方三個檔案是實際可用的版本。

composables/useTheme.ts 
import { ref, watchEffect } from 'vue'

// 全域單例 — 不需要 provide/inject，直接 import 即可
const theme = ref<'light' | 'dark'>(
  (typeof localStorage !== 'undefined' && localStorage.getItem('theme') as any) || 'light'
)

watchEffect(() => {
  document.documentElement.setAttribute('data-theme', theme.value)
  localStorage.setItem('theme', theme.value)
})

export function useTheme() {
  const toggle = () => { theme.value = theme.value === 'dark' ? 'light' : 'dark' }
  return { theme, toggle }
}
composables/useToast.ts 
import { ref } from 'vue'

type Toast = { id: number; type: 'success' | 'error' | 'info'; title: string; msg?: string; state: 'in' | 'out' }

const toasts = ref<Toast[]>([])
let seq = 1

function push(type: Toast['type'], title: string, msg?: string) {
  const id = seq++
  toasts.value.push({ id, type, title, msg, state: 'in' })
  setTimeout(() => {
    const t = toasts.value.find(x => x.id === id)
    if (t) t.state = 'out'
    setTimeout(() => { toasts.value = toasts.value.filter(x => x.id !== id) }, 400)
  }, 3600)
}

export function useToast() {
  return {
    toasts,
    success: (t: string, m?: string) => push('success', t, m),
    error:   (t: string, m?: string) => push('error', t, m),
    info:    (t: string, m?: string) => push('info', t, m),
  }
}
composables/useReadingProgress.ts 
import { ref, onMounted, onBeforeUnmount } from 'vue'

export function useReadingProgress(target?: () => HTMLElement | null) {
  const pct = ref(0)

  const onScroll = () => {
    const el = target?.() ?? null
    if (el) {
      const rect = el.getBoundingClientRect()
      const total = el.offsetHeight - window.innerHeight
      pct.value = Math.max(0, Math.min(100, (-rect.top / total) * 100))
    } else {
      const total = document.documentElement.scrollHeight - window.innerHeight
      pct.value = Math.max(0, Math.min(100, (window.scrollY / total) * 100))
    }
  }

  onMounted(() => { window.addEventListener('scroll', onScroll, { passive: true }); onScroll() })
  onBeforeUnmount(() => window.removeEventListener('scroll', onScroll))

  return { pct }
}
§ tip 關於響應式：Vue 的 ref() 取代 React 的 useState。讀寫時要加 .value，但在 template 裡會自動展開。

components/MobileDrawer.vue 
<script setup lang="ts">
import { watch, onBeforeUnmount } from 'vue'

const props = defineProps<{ open: boolean }>()
const emit = defineEmits<{ (e: 'close'): void; (e: 'navigate', view: string): void }>()

const onKey = (e: KeyboardEvent) => { if (e.key === 'Escape') emit('close') }

watch(() => props.open, (v) => {
  document.body.style.overflow = v ? 'hidden' : ''
  v ? window.addEventListener('keydown', onKey)
    : window.removeEventListener('keydown', onKey)
})
onBeforeUnmount(() => { document.body.style.overflow = ''; window.removeEventListener('keydown', onKey) })

const go = (view: string) => { emit('navigate', view); emit('close') }
</script>

<template>
  <Teleport to="body">
    <div class="drawer-overlay" :class="{ open }" @click="emit('close')" />
    <aside class="drawer" :class="{ open }" :aria-hidden="!open">
      <div class="drawer-head">
        <span>MY BLOG WEB.</span>
        <button class="drawer-close" @click="emit('close')">✕</button>
      </div>
      <nav class="drawer-list">
        <a v-for="v in ['home','articles','search','my']"
           :key="v"
           href="#"
           class="drawer-link"
           @click.prevent="go(v)">
          {{ v }} <span class="arr">→</span>
        </a>
      </nav>
    </aside>
  </Teleport>
</template>
§ 03 — Cheatsheet

React → Vue 3 對照表。

15 mappings
ReactVue 3備註
useState→ref / reactive讀寫加 .value；template 自動 unwrap
useEffect→watchEffect / watch沒有 dependency array，自動追蹤
useMemo→computedlazy + cache，行為一致
useCallback→不需要Vue 模板不會 re-render 整顆 tree
useRef (DOM)→ref(null) + :ref直接 elRef.value 取得 DOM
useContext→provide / inject需要傳 InjectionKey 才有完整型別
className→class支援陣列與物件語法
onClick→@click修飾子 .stop / .prevent 很好用
{cond && <X/>}→v-if="cond"另有 v-show（display:none）
arr.map(...)→v-for + :key:key 必填
style={{...}}→:style="{...}"支援物件、陣列、字串
children prop→<slot/>多個 named slot 直接 name= 區分
Portal→<Teleport to="body">內建，不用第三方
React.lazy→defineAsyncComponentVite 有自動 chunk split
CSS Modules→<style scoped>用 attribute selector 自動 scope
§ 04 — Migration plan

四步搬家。

~2 weeks

用這個順序最不會踩坑。重點是先把 token 與 layout 搬過去，行為層級的東西放最後。

§ 01 · Day 1–2

Tokens 與 reset

把 enhancements.css + 設計系統的 root 變數整包丟進 src/styles/tokens.css。Vue 完全不碰，純 CSS。

§ 02 · Day 3–5

Layout components

先做 TopBar / Footer / SectionFrame。沒互動，純 props + slots。

§ 03 · Day 6–9

Composables

useToast / useTheme / useReadingProgress / useDrawer。每個都附上單元測試。

§ 04 · Day 10–14

Pages 與路由

用 Vue Router 把 prototype 的 7 個 view 拆成 7 個 page。最後做 SSR 與 sitemap。

§ 別做的事 不要一次到位。先讓網站可以 ship，再回頭做 a11y、SEO、performance。每次 refactor 都要有 commit 可以 revert。Migration 最常見的失敗是「重寫太多」。
