命名與檔案結構

一個 composable 就是把 reactive 邏輯包進一個函數，並回傳 ref / computed / 方法。命名上以 use 開頭已經是社群共識，但怎麼分檔、怎麼放各家寫法仍差很多。

use 開頭的慣例

useFoo、useFooBar — 永遠以 use 開頭，camelCase；如果這個 composable 主要在處理某個 store 的子集，可以用 useFooStore() 命名。

// composables/useCounter.ts
export function useCounter(initial = 0) {
  const count = ref(initial)
  const inc = () => count.value++
  return { count, inc }
}

檔案放在哪

我目前的習慣是 src/composables/，每個 composable 一個檔，名字跟 export 的函數一致。不要把多個 composable 塞在同一個檔，除非它們強耦合。

Ref 暴露模式

回傳 ref 還是 readonly(ref)？我的判斷：對外暴露的 ref，預設都應該 readonly。修改的方法另外提供成 named function。
