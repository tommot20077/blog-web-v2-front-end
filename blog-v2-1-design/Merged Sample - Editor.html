在 Vue 3 中設計可組合的 Composables

從零理解 composable 的設計模式

命名與檔案結構

一個 composable 就是把 reactive 邏輯包進函數，並回傳 ref / computed / 方法。慣例是用 use 開頭的命名。

Ref 暴露模式

回傳 ref 還是 readonly(ref)？我的判斷是預設都 readonly，需要改值就提供 method。

// composables/useCounter.ts
export function useCounter(initial = 0) {
  const count = ref(initial)
  const inc = () => count.value++
  return { count: readonly(count), inc }
}
